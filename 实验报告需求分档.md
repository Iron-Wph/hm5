# 模块5最终项目实验报告需求分档与完成方案

## 一、图片内容解析

### 1. 项目主题

本次最终项目通过提交一份实验报告进行考核。报告需要运用或调整课程中讲解的方法，解决一个计算机视觉问题，并对所建模型的成果进行评估。

从图片要求来看，任务核心是：

- 使用 `toy.mp4` 视频作为数据集。
- 使用基于辐射场的算法合成给定场景的新视图。
- 将视频帧划分为训练集、验证集、测试集，比例约为 `60:20:20`。
- 在处理过程中同步定位所有帧，并使场景中的时间变化尽量简要。
- 验证测试视图的合成质量，并判断合成场景是否逼真。

视频下载链接：`https://share.weiyun.com/or5rADfK`

### 2. 报告结构要求

报告必须严格包含以下结构：

1. `Introdução 引言`
   - 问题描述。
   - 文献综述，可选。

2. `Dataset e Tecnologia 数据集与技术`
   - 数据描述。
   - 数据探索。
   - 覆盖区域、场景中可能的重要细节。

3. `Metodologia 方法论`
   - 提出问题的解决方案。
   - 列出使用的模型。
   - 说明流程各阶段可用的参数。

4. `Resultados 结果`
   - 数据可视化与特征分析。
   - 模型结果。
   - 对结果进行讨论和比较。

5. `Conclusões 总结`
   - 关于问题特征的讨论。
   - 关于最佳模型的建议。

### 3. 明确要求

- 至少评估两种不同架构，例如 `NeRF` 和 `3DGS`。
- 探索不同场景或设置，例如模型大小、训练超参数、定位超参数。
- 说明选择评价指标的依据，例如 `PSNR`、`SSIM`、`LPIPS`。
- 对结果进行批判性分析，至少讨论：
  - 是否存在伪影。
  - 新视图的表观质量。
  - 合成结果是否逼真。

## 二、实验目标建议

建议将实验目标写成：

> 本实验基于单段玩具场景视频，通过视频帧提取、相机位姿估计和辐射场建模，完成静态场景的新视图合成。实验比较 NeRF 类方法与 3D Gaussian Splatting 类方法在重建质量、视觉逼真度、训练成本和渲染效率上的差异，并结合 PSNR、SSIM、LPIPS 与定性可视化结果，给出适用于该数据集的模型选择建议。

这个目标覆盖了图片中的全部硬性要求，同时为后续创新实验留下空间。

## 三、推荐完成路线

### 阶段 1：数据准备

1. 下载 `toy.mp4`。
2. 从视频中抽帧，得到图像序列。
3. 进行初步清洗：
   - 去除模糊帧。
   - 去除严重重复帧。
   - 如果视频中存在明显运动干扰，选择相对稳定的片段。
4. 统一图像分辨率，例如 `800px` 或 `960px` 宽度。
5. 按 `60:20:20` 划分训练集、验证集和测试集。

建议不要完全随机划分相邻帧。视频相邻帧非常相似，随机切分容易导致测试结果虚高。更好的做法是先估计相机轨迹，再按照视角覆盖范围分层划分，使测试集包含真正未见过的视角。

### 阶段 2：相机定位

推荐使用 `COLMAP` 完成相机位姿估计：

- 输入抽取的视频帧。
- 进行特征提取与匹配。
- 生成稀疏点云与相机参数。
- 检查相机轨迹是否连续。

如果定位失败，可以尝试：

- 降低抽帧密度，减少重复帧。
- 提升图像清晰度或分辨率。
- 使用顺序匹配模式。
- 删除反光、模糊或运动过大的帧。

### 阶段 3：模型训练

至少训练两类模型：

1. NeRF 类模型
   - 可选工具：Nerfstudio 的 `nerfacto`、Instant-NGP、Mip-NeRF 系列实现。
   - 重点观察：细节还原、颜色一致性、训练稳定性。

2. 3D Gaussian Splatting 类模型
   - 可选工具：GraphDECO 3DGS、gsplat、Nerfstudio 中的 Gaussian Splatting 实现。
   - 重点观察：渲染速度、几何外观、边缘伪影、透明区域表现。

### 阶段 4：多场景或多参数实验

至少做一组参数变化，不要只跑默认配置。建议选择下列方向之一：

- 不同训练迭代次数：例如 `7k / 15k / 30k`。
- 不同输入分辨率：例如 `480p / 720p / 原始分辨率`。
- 不同抽帧间隔：例如每 `3` 帧、每 `5` 帧、每 `10` 帧。
- 不同定位参数：例如顺序匹配与穷举匹配对比。
- 不同模型容量：例如较小模型与较大模型对比。

### 阶段 5：结果评估

必须同时包含定量与定性评估。

定量指标建议：

- `PSNR`：衡量像素级重建误差，数值越高越好。
- `SSIM`：衡量结构相似性，更关注图像结构质量，数值越高越好。
- `LPIPS`：衡量感知相似度，更接近人眼感知，数值越低越好。

定性评估建议：

- 展示测试集真实图像、NeRF 合成图像、3DGS 合成图像。
- 展示误差图或局部放大图。
- 重点分析边缘、反光、遮挡、纹理细节和背景区域。

## 四、需求分档

### A 档：基础可交付

适合时间紧、目标是稳定完成作业的情况。

必须完成：

- 成功下载并处理 `toy.mp4`。
- 抽帧并按 `60:20:20` 划分数据集。
- 使用 COLMAP 或同类工具完成相机位姿估计。
- 至少训练两个模型：
  - 一个 NeRF 类模型。
  - 一个 3DGS 类模型。
- 使用 `PSNR`、`SSIM`、`LPIPS` 进行测试集评估。
- 报告中包含课程要求的五个章节。
- 给出基本对比表格和至少一组可视化结果。

预期报告质量：

- 能满足图片中的硬性要求。
- 分析深度较基础。
- 创新性较弱，但不容易偏题。

### B 档：标准完成

适合想拿到较稳成绩的情况。

在 A 档基础上增加：

- 说明抽帧策略与数据划分原因。
- 展示相机轨迹或稀疏点云结果。
- 对 NeRF 和 3DGS 分别做至少一组参数对比。
- 记录训练时间、显存占用或渲染速度。
- 对每个指标解释其适用性与局限性。
- 使用表格比较：
  - 平均 PSNR。
  - 平均 SSIM。
  - 平均 LPIPS。
  - 训练时间。
  - 渲染速度。
  - 主要伪影类型。

预期报告质量：

- 不只是“跑出结果”，而是能解释为什么某个模型更好。
- 结果讨论更完整，符合课程项目的正式报告要求。

### C 档：高分增强

适合有较多时间、希望报告明显优于普通提交的情况。

在 B 档基础上增加：

- 使用视角分层的数据划分，而不是简单随机划分。
- 增加数据清洗策略，例如去模糊、去重复、背景遮罩。
- 对失败案例进行分析，例如：
  - 物体边缘漂浮。
  - 背景重建不稳定。
  - 反光区域失真。
  - 视角外推时出现空洞。
- 加入误差热力图或局部放大对比。
- 从质量、速度、训练成本三个角度综合推荐最佳模型。
- 在结论中明确说明该场景更适合 NeRF 还是 3DGS，并给出原因。

预期报告质量：

- 具备比较完整的实验设计。
- 能体现对数据、模型和指标的批判性理解。
- 可作为较成熟的课程项目报告。

### D 档：创新加分

适合希望报告具有“创新性质”的情况。创新不一定要发明新模型，可以是实验设计、分析方式或评估方法上的改进。

可选创新方向：

1. 轨迹感知划分
   - 先根据相机位姿估计轨迹，再按空间位置划分训练集、验证集和测试集。
   - 目的：避免相邻帧泄漏，提高测试集可信度。

2. 自动帧质量筛选
   - 根据模糊度、亮度、重复度筛掉低质量帧。
   - 比较“原始帧训练”和“清洗帧训练”的结果差异。

3. 误差热力图分析
   - 对测试图像生成像素误差图。
   - 分析错误集中在哪些区域，例如物体边缘、纹理区域或背景。

4. 低资源训练实验
   - 只使用 `25% / 50% / 75% / 100%` 训练帧。
   - 比较 NeRF 和 3DGS 在少量数据下的鲁棒性。

5. 速度-质量权衡曲线
   - 横轴使用训练时间或渲染速度。
   - 纵轴使用 PSNR、SSIM 或 LPIPS。
   - 讨论哪个模型在有限计算资源下更划算。

6. 背景遮罩或前景聚焦评估
   - 对玩具主体区域单独计算指标。
   - 避免背景占比过大导致整体指标掩盖主体质量。

7. 新视角视频展示
   - 在测试视角之间插值生成一段环绕视频。
   - 用于直观展示模型是否真正学到了三维结构。

建议选择 `2` 到 `3` 个创新点即可。过多创新点会分散报告重点。

## 六、报告写作模板

### 1. 引言

建议写清楚：

- 新视图合成是什么问题。
- 为什么从视频中重建三维场景有意义。
- 本实验为什么选择 NeRF 和 3DGS 对比。
- 本实验要回答的问题：
  - 哪种方法在 `toy.mp4` 上重建质量更高？
  - 哪种方法渲染速度更快？
  - 哪种方法更容易出现伪影？
  - 数据清洗或参数变化是否能提升结果？

### 2. 数据集与技术

建议包含：

- 视频来源。
- 视频时长、分辨率、帧率。
- 抽帧数量。
- 训练、验证、测试数量。
- 场景特点：
  - 主体是否清晰。
  - 背景是否复杂。
  - 光照是否稳定。
  - 是否存在反光、遮挡或运动模糊。
- 使用的工具：
  - 抽帧工具。
  - COLMAP。
  - NeRF 训练框架。
  - 3DGS 训练框架。
  - 评价脚本。

### 3. 方法论

建议按照流程图式描述：

1. 视频抽帧。
2. 帧清洗与缩放。
3. 相机位姿估计。
4. 数据集划分。
5. NeRF 训练。
6. 3DGS 训练。
7. 测试视图渲染。
8. 计算 PSNR、SSIM、LPIPS。
9. 定性和定量结果比较。

需要说明的参数：

- 抽帧间隔。
- 图像分辨率。
- 训练迭代次数。
- 学习率或默认训练配置。
- COLMAP 匹配方式。
- 测试集选取方式。

### 4. 结果

建议至少包含以下表格：

| 模型 | PSNR ↑ | SSIM ↑ | LPIPS ↓ | 训练时间 | 渲染速度 | 主要问题 |
| --- | --- | --- | --- | --- | --- | --- |
| NeRF | 待填 | 待填 | 待填 | 待填 | 待填 | 待填 |
| 3DGS | 待填 | 待填 | 待填 | 待填 | 待填 | 待填 |

建议至少包含以下图像：

- 原始测试图像。
- NeRF 渲染图像。
- 3DGS 渲染图像。
- 局部放大对比。
- 可选：误差热力图。
- 可选：相机轨迹图。

结果讨论可以从以下角度展开：

- 哪个模型整体指标更好。
- 哪个模型视觉效果更自然。
- 哪个模型更容易出现伪影。
- 指标与人眼观察是否一致。
- 参数变化是否带来显著提升。
- 训练成本和效果是否匹配。

### 5. 总结

总结中建议给出明确结论：

- 如果 3DGS 质量接近 NeRF 且速度明显更快，可以推荐 3DGS 作为该场景的首选方法。
- 如果 NeRF 在细节、稳定性或感知质量上明显更好，可以说明 NeRF 更适合该小场景精细重建。
- 如果两者各有优劣，应从应用需求出发给出建议：
  - 追求实时渲染：优先 3DGS。
  - 追求稳定细节：优先 NeRF。
  - 计算资源有限：选择训练更快或参数更少的方法。

## 八、最推荐的执行方案

如果只选择一套最稳妥的方案，建议如下：

1. 从 `toy.mp4` 抽帧，去除模糊帧。
2. 使用 COLMAP 估计相机位姿。
3. 按相机轨迹分层划分 `60:20:20` 数据集。
4. 训练 `Nerfacto/Instant-NGP` 作为 NeRF 类基线。
5. 训练 `3D Gaussian Splatting` 作为 3DGS 基线。
6. 做一组训练迭代次数或输入分辨率对比。
7. 使用 `PSNR`、`SSIM`、`LPIPS` 评估。
8. 增加一个创新分析：推荐“自动帧质量筛选”或“误差热力图”。
9. 在报告中从质量、速度、伪影、资源消耗四个角度推荐最佳模型。

该方案工作量适中，覆盖硬性要求，也能体现一定创新性。
