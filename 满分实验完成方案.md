# 新视图合成满分实验完成方案

## 一、总体定位

本实验建议做成“全流程编程化的新视图合成实验流水线”。模型核心使用成熟开源实现，创新性主要体现在数据处理、数据划分、评价维度和误差分析上。

推荐技术路线：

- 数据处理：`Python + OpenCV`
- 视频抽帧：`OpenCV` 或 `ffmpeg`
- 相机位姿估计：`COLMAP` 或 `PyCOLMAP`
- NeRF 模型：`Nerfstudio - nerfacto`
- 3DGS 模型：`Nerfstudio - splatfacto`
- 指标评价：`PSNR / SSIM / LPIPS`
- 可视化：`matplotlib / seaborn / pandas`

这样可以满足作业要求中的：

- 使用 `toy.mp4` 数据集。
- 至少比较两种架构：NeRF 和 3DGS。
- 划分训练集、验证集、测试集，比例约为 `60:20:20`。
- 使用指标评价结果。
- 分析伪影、新视图质量和模型差异。
- 体现创新性。

## 二、为什么不建议从零写完整 NeRF 和 3DGS

NeRF 可以写简化版，但完整高质量实现涉及光线采样、体渲染、加速结构、位置编码、训练稳定性和大量调参。

3DGS 更复杂，核心依赖可微高斯光栅化，通常需要 C++/CUDA 加速。从零实现完整 3DGS 不适合作为本课程实验主线。

因此推荐策略是：

> 使用 Nerfstudio 中成熟的 `nerfacto` 和 `splatfacto` 作为模型实现；自己编写数据处理、数据划分、训练调度、指标评估、图表生成和报告分析脚本。

这样既能体现编程能力，也能保证实验可完成、可复现、可比较。

## 五、完整实验流程

### 阶段 1：视频抽帧与帧质量评分

目标：

- 从 `toy.mp4` 中抽取图像帧。
- 为每一帧计算质量指标。
- 为后续自动筛选提供依据。

建议脚本：

```text
scripts/00_extract_score_frames.py
```

建议计算指标：

- 模糊度：Laplacian 方差。
- 亮度：灰度均值。
- 对比度：灰度标准差。
- 重复度：感知哈希或相邻帧 SSIM。

示例逻辑：

```python
blur_score = variance_of_laplacian(gray)
brightness = gray.mean()
contrast = gray.std()
```

输出文件：

```text
data/frames_raw/
data/splits/frame_quality.csv
```

创新性体现：

> 本实验不是简单等间隔抽帧，而是对每一帧进行清晰度、曝光和重复度评分，为后续数据清洗提供量化依据。

### 阶段 2：自动帧筛选

目标：

- 删除低质量帧。
- 减少重复帧。
- 保留视角覆盖充分的帧。

建议脚本：

```text
scripts/01_filter_frames.py
```

筛选规则建议：

- 模糊度低于第 `20%` 分位数的帧删除。
- 亮度过暗或过亮的帧删除。
- 与前一保留帧感知哈希距离过小的帧删除。
- 最终保留帧数控制在 `80-200` 张，视视频长度和显存情况调整。

输出文件：

```text
data/frames_clean/
data/splits/clean_frame_list.csv
```

创新性体现：

> 自动帧筛选可以减少模糊帧和重复帧对三维重建的干扰，提高相机位姿估计稳定性，也能让训练集更紧凑。

### 阶段 3：相机位姿估计

目标：

- 使用 COLMAP 对清洗后的图像进行相机定位。
- 生成稀疏点云、相机参数和 `transforms.json`。

推荐方式：

```bash
ns-process-data images \
  --data data/frames_clean \
  --output-dir data/toy_ns \
  --matching-method sequential
```

说明：

- 视频帧是连续序列，优先使用 sequential matching。
- 如果定位失败，可尝试 exhaustive matching。
- 如果帧过密或重复度高，先降低抽帧密度。

输出文件：

```text
data/toy_ns/transforms.json
data/toy_ns/images/
data/toy_ns/colmap/
```

报告中应展示：

- 稀疏点云截图。
- 相机轨迹图。
- 成功定位的图像数量。

### 阶段 4：轨迹感知数据划分

目标：

- 按 `60:20:20` 划分训练集、验证集、测试集。
- 避免视频相邻帧随机划分造成测试泄漏。

建议脚本：

```text
scripts/03_pose_stratified_split.py
```

方法：

1. 读取 `transforms.json`。
2. 提取每张图像的相机中心位置。
3. 根据相机轨迹顺序或空间位置分层。
4. 每个轨迹区间按比例选择训练、验证、测试帧。
5. 输出 `split.csv`，并修改或生成带 split 标记的 `transforms.json`。

推荐比例：

```text
train: 60%
val: 20%
test: 20%
```

创新性体现：

> 轨迹感知划分比随机划分更严格，可以避免相邻帧过于相似导致测试指标虚高，使测试集更接近真实的新视角合成任务。

### 阶段 5：训练 NeRF 模型

使用 Nerfstudio 的 `nerfacto` 作为 NeRF 类模型。

基础实验：

```bash
ns-train nerfacto \
  --max-num-iterations 15000 \
  --data data/toy_ns \
  nerfstudio-data --eval-mode filename
```

增强实验：

```bash
ns-train nerfacto-big \
  --max-num-iterations 15000 \
  --data data/toy_ns \
  nerfstudio-data --eval-mode filename
```

可比较参数：

- `nerfacto` vs `nerfacto-big`
- `7000 / 15000 / 30000` 迭代次数
- 不同输入分辨率
- 是否使用清洗帧

报告中应记录：

- 训练时间。
- 最终指标。
- 渲染速度。
- 主要伪影。

### 阶段 6：训练 3DGS 模型

使用 Nerfstudio 的 `splatfacto` 作为 3D Gaussian Splatting 类模型。

基础实验：

```bash
ns-train splatfacto \
  --max-num-iterations 15000 \
  --data data/toy_ns \
  nerfstudio-data --eval-mode filename
```

正则化实验：

```bash
ns-train splatfacto \
  --max-num-iterations 15000 \
  --pipeline.model.use_scale_regularization=True \
  --data data/toy_ns \
  nerfstudio-data --eval-mode filename
```

高质量实验：

```bash
ns-train splatfacto \
  --max-num-iterations 15000 \
  --pipeline.model.cull_alpha_thresh=0.005 \
  --pipeline.model.continue_cull_post_densification=False \
  --data data/toy_ns \
  nerfstudio-data --eval-mode filename
```

可比较参数：

- 默认 `splatfacto`
- scale regularization
- alpha cull threshold
- 迭代次数
- 输入分辨率

报告中应重点分析：

- 边缘是否出现漂浮高斯。
- 是否有尖刺状伪影。
- 渲染速度是否明显优于 NeRF。
- 视觉质量与指标是否一致。

## 七、核心创新点设计

### 创新点 1：自动帧质量筛选

问题：

视频中可能存在模糊、重复、曝光异常帧，这些帧会影响 COLMAP 定位和模型训练。

方法：

- 用 Laplacian 方差检测模糊。
- 用亮度均值检测曝光异常。
- 用感知哈希或相邻帧 SSIM 检测重复帧。

报告写法：

> 为了提升重建质量，本实验在抽帧后引入自动帧质量筛选机制，从清晰度、曝光和重复度三个维度筛除低质量帧。该方法减少了低质量帧对相机定位和辐射场训练的干扰。

### 创新点 2：轨迹感知数据划分

问题：

视频相邻帧高度相似，随机划分训练集和测试集会导致测试结果虚高。

方法：

- 从 `transforms.json` 提取相机中心。
- 沿相机轨迹分段。
- 在不同轨迹段中按比例采样训练、验证和测试帧。

报告写法：

> 与随机划分不同，本实验采用轨迹感知的数据划分策略，使测试帧在视角空间上相对独立，从而更真实地评估模型的新视图合成能力。

### 创新点 3：主体区域评价

问题：

全图指标容易被背景区域主导，无法准确反映玩具主体的重建质量。

方法：

- 使用 GrabCut、简单阈值或手动初始框生成主体遮罩。
- 分别计算全图指标和主体区域指标。

报告写法：

> 除全图 PSNR、SSIM 和 LPIPS 外，本实验额外计算玩具主体区域的评价指标，以观察模型是否真正重建了核心对象，而不是仅依赖背景区域获得较高分数。

### 创新点 4：误差热力图与失败案例分类

问题：

单一数值指标无法说明模型具体失败在哪里。

方法：

- 对真实图像和渲染图像做像素差。
- 生成误差热力图。
- 对错误区域进行分类分析。

分析类别：

- 物体边缘伪影。
- 反光区域失真。
- 纹理细节丢失。
- 背景漂浮或模糊。
- 视角外推时的空洞。

报告写法：

> 误差热力图显示，模型误差主要集中在物体边缘和高频纹理区域，说明该场景的几何边界和细节纹理是影响新视图质量的主要因素。

### 创新点 5：速度-质量权衡分析

问题：

只比较图像质量不够完整，实际应用还关心训练成本和渲染速度。

方法：

- 记录训练时间。
- 记录渲染 FPS 或单张图像渲染时间。
- 绘制速度-质量曲线。

报告写法：

> 本实验不仅比较重建质量，还比较模型的训练成本和渲染效率。该分析能够说明 NeRF 和 3DGS 在实际应用中的取舍。

## 九、最终结论建议写法

可根据实际结果调整，但推荐结论框架如下：

> 实验结果表明，对于 `toy.mp4` 这类小型静态物体场景，3DGS 类方法通常在训练速度和渲染效率上具有明显优势，适合快速生成交互式新视图；NeRF 类方法在部分细节连续性和颜色稳定性上具有一定优势。通过自动帧质量筛选，可以减少模糊帧和重复帧对相机位姿估计的影响；通过轨迹感知数据划分，可以降低相邻帧泄漏造成的测试指标虚高；通过主体区域评价和误差热力图，可以更细致地解释模型失败区域。综合图像质量、视觉真实感、训练成本和渲染效率，本实验最终推荐在该场景中优先使用实际结果表现最优的模型。

9. 生成误差热力图和速度-质量曲线。
10. 在报告中从质量、速度、伪影、资源消耗和创新策略有效性五个角度做综合分析。

这套方案覆盖作业全部硬性要求，也具备清晰的创新点，适合作为满分目标方案。
