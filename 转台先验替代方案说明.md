# 转台先验替代方案说明

## 为什么要换方法

当前 `COLMAP` 路线失败：即使使用 `exhaustive` matching，日志仍显示：

```text
Starting with 300 images
Colmap matched 3 images
COLMAP only found poses for 1.00% of the images
```

这说明 COLMAP 无法从该视频中稳定估计相机位姿。继续训练会得到只有几张图的无效模型，不能用于实验报告。

该现象常见于“相机固定、物体在转台上旋转”的视频。COLMAP 假设场景静态、相机运动；如果实际是物体运动，COLMAP 会把它视为非静态场景，因此匹配失败。

## 新方法

使用“转台/圆轨迹先验位姿”：

> 将物体绕自身旋转等价转换为相机绕静态物体旋转，手动生成圆形相机轨迹位姿。

这样不再依赖 COLMAP，而是直接生成 Nerfstudio 可用的 `transforms.json`。

## 使用方式

进入服务器项目目录：

```bash
cd /mnt/public2/wph/codes/AI/one_click_nvs
chmod +x run_turntable_pipeline.sh
```

如果已经有 `data/frames_clean`，可以直接运行：

```bash
./run_turntable_pipeline.sh
```

如果想强制重新训练：

```bash
./run_turntable_pipeline.sh --force-retrain
```

如果只想先生成转台位姿数据，不训练：

```bash
./run_turntable_pipeline.sh --skip-train --skip-eval
```

## 报告中如何解释创新性

可以写：

> 由于原始视频中相机运动不明显，且场景可能更接近物体转台旋转，传统 COLMAP/SfM 位姿估计只成功注册极少数帧，无法满足 NeRF 与 3DGS 的训练要求。因此，本实验设计了一个 object-centric turntable pose prior：将物体旋转等价为相机绕物体进行圆轨迹运动，并根据视频帧序列生成相机外参。该策略避免了 COLMAP 对静态场景和充分纹理特征的依赖，使该类转台视频仍可用于新视图合成实验。

这本身就是一个合理创新点：**当 COLMAP 失败时，使用对象中心的先验几何假设完成位姿初始化。**

## 风险和局限

- 圆轨迹位姿是假设，不是真实相机位姿。
- 如果视频不是完整一圈，需要调整 `total_angle_degrees`。
- 如果物体没有居中，可能需要裁剪或重新居中。
- 该方案更适合转台视频，不适合复杂自由运动视频。

配置位置：

```json
"turntable": {
  "fov_degrees": 55.0,
  "radius": 3.0,
  "elevation_degrees": 8.0,
  "total_angle_degrees": 360.0
}
```

如果新视图变形严重，优先调整：

- `total_angle_degrees`
- `radius`
- `elevation_degrees`
- 是否对图像进行主体裁剪

## 参考依据

Nerfstudio 的 nerfstudio dataparser 支持 `transforms.json` 中显式提供 `train_filenames / val_filenames / test_filenames`，也支持在缺少 SfM 点云时对 Gaussian Splatting 使用随机点云初始化。相关实现见 Nerfstudio 官方代码：
