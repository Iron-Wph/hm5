# 一键完成式实验执行说明

本文档对应目录：`one_click_nvs/`

目标是把 `toy.mp4` 新视图合成实验做成一条尽量自动化的流水线：抽帧、筛帧、COLMAP 位姿估计、轨迹感知划分、训练 NeRF/3DGS、评价指标、生成图表和报告草稿。

## 1. 你需要先准备什么

### 必须准备

1. 下载 `toy.mp4`。
2. 放到以下位置：

```text
one_click_nvs/data/raw/toy.mp4
```

### 推荐硬件

- NVIDIA GPU。
- 显存建议至少 `8GB`，更推荐 `12GB+`。
- 如果显存不足，先把 `one_click_nvs/config.json` 中的 `max_width` 调小，例如 `640`。

### 推荐系统

优先推荐：

- Windows + WSL2 Ubuntu 22.04
- 或原生 Linux

Windows 原生也可以，但安装 CUDA 编译依赖时更容易出问题。

## 2. 安装环境

### 2.1 创建 Conda 环境

```bash
conda create -n toy_nvs python=3.8 -y
conda activate toy_nvs
python -m pip install --upgrade pip
```

### 2.2 安装 PyTorch CUDA

```bash
pip install torch==2.1.2+cu118 torchvision==0.16.2+cu118 --extra-index-url https://download.pytorch.org/whl/cu118
conda install -c "nvidia/label/cuda-11.8.0" cuda-toolkit -y
```

### 2.3 安装 Nerfstudio 和 3DGS 后端

```bash
pip install ninja
pip install git+https://github.com/NVlabs/tiny-cuda-nn/#subdirectory=bindings/torch
pip install nerfstudio
pip install gsplat
```

### 2.4 安装本实验脚本依赖

在 `one_click_nvs` 目录中执行：

```powershell
python -m pip install -r requirements.txt
```

或使用脚本安装辅助依赖：

```powershell
cd one_click_nvs
.\run_pipeline.ps1 -Install
```

注意：`-Install` 只安装 Python 辅助包，不自动安装 CUDA/Nerfstudio 的核心编译依赖。

### 2.5 安装 FFmpeg 和 COLMAP 命令行工具

`ns-process-data` 需要能在命令行中找到 `ffmpeg`。如果没有安装，会在 COLMAP/Nerfstudio 数据处理阶段报错。

Ubuntu/Linux 推荐：

```bash
conda install -c conda-forge ffmpeg colmap -y
```

如果使用系统包管理器，也可以：

```bash
sudo apt update
sudo apt install -y ffmpeg colmap
```

安装后检查：

```bash
ffmpeg -version
colmap -h
```

## 4. 每一步会做什么

### Step 1：抽帧与帧质量评分

脚本：

```text
scripts/00_extract_score_frames.py
```

输出：

```text
data/frames_raw/
data/splits/frame_quality.csv
data/splits/video_metadata.json
```

作用：

- 从 `toy.mp4` 抽帧。
- 计算模糊度、亮度、对比度和感知哈希。

创新点：

- 不只是简单抽帧，而是为每帧建立质量评分。

### Step 2：自动帧筛选

脚本：

```text
scripts/01_filter_frames.py
```

输出：

```text
data/frames_clean/
data/splits/clean_frame_list.csv
data/splits/rejected_frame_list.csv
data/splits/filter_summary.json
```

作用：

- 删除模糊帧。
- 删除曝光异常帧。
- 删除重复度太高的帧。
- 控制最大训练帧数。

创新点：

- 自动清洗数据，降低低质量帧对 COLMAP 和模型训练的影响。

### Step 3：COLMAP / Nerfstudio 数据处理

脚本：

```text
scripts/02_process_colmap.py
```

核心命令：

```bash
ns-process-data images --data data/frames_clean --output-dir data/toy_ns --matching-method sequential
```

输出：

```text
data/toy_ns/transforms.json
data/toy_ns/images/
data/toy_ns/colmap/
```

作用：

- 进行相机位姿估计。
- 生成 NeRF/3DGS 可训练数据。

### Step 4：轨迹感知划分

脚本：

```text
scripts/03_pose_stratified_split.py
```

输出：

```text
data/splits/split.csv
data/splits/split_summary.json
data/toy_ns/transforms_trainval.json
data/toy_ns/transforms_traintest.json
```

作用：

- 生成 `60:20:20` 的训练、验证、测试划分。
- 默认训练最终模型时使用 `transforms_traintest.json`，因此 Nerfstudio 的 eval split 对应测试集。

创新点：

- 避免视频相邻帧随机划分导致测试指标虚高。

### Step 5：训练 NeRF 和 3DGS

脚本：

```text
scripts/04_train_all.py
```

默认训练模型：

- `nerfacto_base`
- `nerfacto_big`
- `splatfacto_base`
- `splatfacto_scale_reg`

输出：

```text
outputs/
results/run_manifest.json
```

作用：

- 自动调用 `ns-train`。
- 记录每个模型的训练耗时和 `config.yml` 路径。

### Step 6：模型评价

脚本：

```text
scripts/05_eval_metrics.py
```

输出：

```text
results/模型名/eval.json
results/模型名/renders/
results/tables/metrics_summary.csv
results/tables/metrics_detail.csv
```

作用：

- 调用 `ns-eval`。
- 汇总 PSNR、SSIM、LPIPS 等指标。
- 如果渲染结果中能找到真实图和预测图配对，会额外计算自定义 PSNR/SSIM。

### Step 7：生成图表

脚本：

```text
scripts/06_make_figures.py
```

输出：

```text
results/figures/frame_quality_distribution.png
results/figures/camera_trajectory_split.png
results/figures/psnr_barplot.png
results/figures/ssim_barplot.png
results/figures/lpips_barplot.png
results/figures/speed_quality_tradeoff.png
```

作用：

- 生成报告中可直接使用的图。

### Step 8：生成报告草稿

脚本：

```text
scripts/07_write_report_outline.py
```

输出：

```text
report/final_report_outline.md
```

作用：

- 自动汇总实验流程、创新点、指标表和图表引用。
- 后续可在此基础上补充文字分析和最终结论。

## 5. 推荐满分执行顺序

第一轮：只验证数据处理能跑通。

```powershell
.\run_pipeline.ps1 -SkipTrain -SkipEval
```

Ubuntu/Linux：

```bash
./run_pipeline.sh --skip-train --skip-eval
```

第二轮：确认 COLMAP 位姿估计正常后，跑完整实验。

```powershell
.\run_pipeline.ps1
```

Ubuntu/Linux：

```bash
./run_pipeline.sh
```

第三轮：如果显存不足，修改 `config.json`：

```json
"max_width": 640
```

并可减少模型，只保留：

- `nerfacto_base`
- `splatfacto_base`
- `splatfacto_scale_reg`

## 6. 满分报告中要强调的创新性

建议在报告中明确写：

1. 本实验实现了从视频到模型评价的全流程编程化流水线。
2. 使用自动帧质量筛选减少模糊帧和重复帧影响。
3. 使用轨迹感知划分，避免相邻帧泄漏造成指标虚高。
4. 同时比较 NeRF 和 3DGS 的质量、速度、伪影和资源消耗。
5. 通过误差图、主体区域评价和速度-质量曲线增强批判性分析。

## 7. 常见问题

### 找不到 `toy.mp4`

把视频放到：

```text
one_click_nvs/data/raw/toy.mp4
```

### 找不到 `ns-process-data`

说明 Nerfstudio 没有安装好，或 Conda 环境没有激活。

检查：

```bash
conda activate toy_nvs
ns-process-data --help
```

### COLMAP 定位失败

尝试：

- 把 `config.json` 中的 `frame_step` 调大，例如从 `5` 改为 `8`。
- 把 `max_frames` 降低，例如 `120`。
- 修改 `matching_method` 为 `exhaustive`。

如果 `data/toy_ns/transforms.json` 中只注册了极少图像，例如：

```bash
python - <<'PY'
import json
d=json.load(open('data/toy_ns/transforms.json'))
print(len(d['frames']))
PY
```

输出小于 `20`，不要继续训练。应先删除或备份旧的 `data/toy_ns`，改用 exhaustive matching 重新生成位姿：

```bash
mv data/toy_ns data/toy_ns_bad_$(date +%Y%m%d_%H%M%S)
python scripts/02_process_colmap.py --config config.json
python scripts/03_pose_stratified_split.py --config config.json
```

确认 split 中 train/val/test 都非空后，再强制重新训练：

```bash
python scripts/04_train_all.py --config config.json --force-retrain
```

### 显存不足

尝试：

- 把 `max_width` 改为 `640`。
- 先只训练 `nerfacto_base` 和 `splatfacto_base`。
- 把 `max_num_iterations` 改为 `7000`。

### 看到 `Training Finished` 但没有继续下一个模型

这通常表示当前 Nerfstudio 模型已经训练完成，但 viewer 还在运行，外层一键脚本仍在等待这个进程退出。

本项目新版 `scripts/04_train_all.py` 已加入：

```text
--viewer.quit-on-train-completion True
```

训练完成后会自动退出 viewer，并继续训练后续模型。如果你是在旧脚本上遇到这个情况，可以按一次 `Ctrl+C` 停掉当前 viewer，然后更新脚本后重新运行：

```bash
./run_pipeline.sh
```

脚本会检测已有的 `outputs/模型名/方法名/*/config.yml`，默认跳过已完成模型，继续训练剩余模型。

### `ns-eval` 报错但已经生成 `metrics_summary.csv`

部分 Nerfstudio 版本在执行 `ns-eval` 时可能出现 datamanager 或空 eval 列表兼容问题，例如：

```text
fixed_indices_eval_dataloader
IndexError: list index out of range
```

这通常不表示训练失败。只要已经存在多个 `outputs/*/*/*/config.yml`，说明模型已经训练完成。

如果只需要继续生成报告资产，可以跳过官方 `ns-eval`，改用训练日志中的指标：

```bash
python scripts/05_eval_metrics.py --config config.json --skip-ns-eval
python scripts/06_make_figures.py --config config.json
python scripts/07_write_report_outline.py --config config.json
```

然后检查：

```bash
cat results/tables/metrics_summary.csv
```

### PowerShell 不允许运行脚本

在当前 PowerShell 中执行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

然后重新运行：

```powershell
.\run_pipeline.ps1
```
